Design a mobile application prototype for children aged 10–12 that teaches disaster preparedness and survival skills through an interactive gamified experience.

The app should use a playful, cartoon-style interface inspired by Japanese doodle illustration, with soft pastel colors, friendly icons, rounded UI components, and storytelling visuals.

The app must feel safe, playful, and educational, not scary or realistic.

Target: Southeast Asian students

Platform: Mobile App (Vertical 9:16)

Overall Design Style

Visual Style:

Hand-drawn Japanese doodle illustration

Storybook-like UI

Cartoon disaster representation (no realistic destruction)

Soft pastel color palette

Friendly characters

Animated stickers and icons

Large rounded buttons

Clear simple typography

UI Principles:

Kid-friendly

Minimal text

Interactive storytelling

Gamified learning

High visual engagement

App Structure

The app contains 4 main sections

1️⃣ Dashboard (Home Base)
2️⃣ Wira Atlas (Interactive Survival Map)
3️⃣ AR Time Machine (Real World AR Training)
4️⃣ Possum Protocol (Offline Survival System)

Bottom Navigation Bar:
Home | Atlas | AR | Survival

Design a friendly command center dashboard.

Purpose:

UI Layout:

“Hello Explorer! Ready to train today?”

Progress level indicator

Card 1:

Icon: magnifying glass

Card 3:
Emergency Survival Challenge
Icon: shield badge

Each card contains:

colorful illustration

short description

“Start Mission” button

Next Section – Safety Knowledge Progress

Gamification elements:

Survival badges

Level progression bar

Example badges:
Flood Explorer
Storm Spotter

Bottom Section – Quick Alerts

Small notification cards showing:

Local weather alerts

Safety tips

Training reminders

2️⃣ Module One: Wira Atlas (Interactive Survival Map)

Purpose:
Teach disaster awareness using a cartoon world map survival sandbox.

The map should NOT look like Google Maps.

Instead create:

A stylized illustrated world map with:

cartoon mountains

playful rivers

tiny villages

forests

islands

volcanoes

storms drawn like doodles

The map feels like a storybook adventure map.

Map Interaction

Users can:

zoom

pan

tap locations

discover history

view risk forecasts

Feature 1: Disaster History Archive (Tap to Learn)

Across the map are colorful “memory capsule” icons.

Each capsule represents a past disaster event.

Examples:
Flood
Typhoon
Earthquake
Landslide

Interaction:

When a user taps a location:

A story card slides up from the bottom.

The card looks like a page from a children's picture book.

Disaster Story Card Content

Illustrated animation showing the event.

Example story text:

“During 2021, this village experienced three days of heavy rain.
The flood water became as tall as two elephants!”

Illustrations show:

cartoon rain clouds

smiling elephants showing height comparison

villagers moving to safe shelter

No destruction imagery.

Educational Quiz

After the story, a mini quiz appears.

Example:

“Where should we go during a flood?”

Options:
A – Stay near the river
B – Move to higher ground
C – Hide in the house basement

Correct answer gives:

experience points

survival badge

Confetti animation appears.

Feature 2: Hyper-Local Forecast Mode

Add a toggle dial on the map interface.

Switch modes:

History Mode
Forecast Mode

When Forecast Mode is activated:

The map shows future risk predictions (next 24 hours).

Forecast Visualization

Instead of traditional warning colors, use:

Watercolor-style overlays.

Examples:

Blue watercolor – heavy rain
Wind swirl doodles – strong wind
Mountain cracks – landslide risk
Typhoon swirl icon – storm path

This creates a soft artistic risk visualization.

Safety Guidance

When a user taps a risk zone:

A calm safety card appears.

Example message:

“Strong winds may happen here tomorrow.
Please check that windows are closed.”

Safe Route Feature

The map automatically draws:

A high-contrast safety route line from the user’s location to:

nearest safe shelter

evacuation point

school safety zone

The route is illustrated like a bright adventure path.

3️⃣ Module Two: AR Time-Machine

Purpose:

Turn knowledge into real-world survival training.

This feature uses the phone camera.

AR Gameplay

When activated:

The camera view opens.

The system overlays cartoon survival objects in the real world.

Examples:

First Aid Kit
Emergency Backpack
Safe Shelter Zone
Radio Device

These appear like hidden treasures.

AR Mission Example

Mission Title:

“Find the Emergency Kit”

Instructions:

“Look around your classroom or house to find the hidden emergency supplies.”

Children move their phone to search.

When an item is found:

A sparkle animation appears.

The child taps the item to collect it.

Learning Card

After collecting the object, a quick educational card appears.

Example:

First Aid Kit

Contains:
Bandages
Antiseptic wipes
Emergency whistle

The child receives points and a badge.

4️⃣ Module Three: Possum Protocol (Extreme Survival Mode)

Purpose:

Provide emergency communication tools when disasters destroy networks.

This module demonstrates advanced technical capabilities.

Feature 1: Ultra Power Saving Mode

Interface switches to minimal dark emergency UI.

Only essential features remain:

Flashlight
Emergency broadcast
Battery monitor

Feature 2: Sound Rescue Signal

The phone listens for distress whistle patterns.

If detected:

The system alerts nearby users.

Visual UI shows:

Sound wave animation
Signal strength indicator

Feature 3: Offline Mesh Network

Devices automatically connect with nearby phones.

Features:

Send SOS message
Share GPS location
Receive rescue signals

The UI shows nearby devices as small connected nodes forming a mesh network map.
